All four failure modes — and their recovery

Runtime crash (render error)
Extension throws during React render or event handler.
→ Error boundary catches it. Slot shows error UI. Status bar item turns amber. "Reload Extension" button re-runs dynamic import with cache-bust.
Load failure (bad manifest)
Extension's index.tsx throws on import, or manifest is missing required fields.
→ Extension skipped in registry. Activity rail icon shows ⚠️. Error surfaced in a "Failed extensions" section of the command palette.
IPC timeout
Main process doesn't respond to an IPC call within 10s (frozen, busy).
→ shell.ipc.* rejects with a typed IPCTimeoutError. Extension handles it. No silent hang.
Renderer crash (rare)
Entire Chromium renderer process crashes (memory, native crash).
→ Electron automatically relaunches the renderer. Shell restores previous layout and re-discovers extensions. Extension stores rehydrated from disk.
