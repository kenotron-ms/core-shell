Section 3 (updated) — Extension as npm Package

Extensions are real packages. Proper deps, proper versioning, publishable to any registry.

Three ways an extension lives in the shell

📁

Local (dev)

Drop a folder into extensions/. Has its own package.json. Shell discovers it on startup. No install step — instant iteration.

extensions/
└─ my-canvas/
   ├─ package.json
   ├─ index.tsx
   └─ ...
📦

Installed (npm)

Install any npm package that has "core-shell-extension" in its keywords. Shell discovers it in node_modules/ automatically.

npm install @myorg/amplifier-canvas
# or: core-shell install @myorg/amplifier-canvas
# → restart → it's there
🏗

Bundled (first-party)

Listed directly in shell's package.json as a workspace dep. Built together with the shell. Zero install step for end users — ships inside the app binary.

// shell/package.json
"workspaces": [
  "packages/shell",
  "packages/ext-amplifier-canvas"  ← bundled
]

What the extension's package.json looks like

packages/ext-amplifier-canvas/package.json
{
  "name": "@myorg/amplifier-canvas",
  "version": "0.1.0",
  "type": "module",
  "main": "dist/index.js",
  "keywords": ["core-shell-extension"],

  "peerDependencies": {
    "react": "^18",
    "core-shell": "^1"
  },

  "dependencies": {
    "@xterm/xterm": "^6.0.0",
    "chokidar":     "^4.0.0"
  },

  "devDependencies": {
    "vite": "^5",
    "typescript": "^5"
  }
}

Peer deps (React, core-shell) — provided by the shell at runtime, not bundled into the extension.
Regular deps (xterm, chokidar, etc.) — bundled into the extension's dist. Extension is self-contained.

Discovery order on startup

1️⃣
Bundled (workspace packages)
Extensions in the shell's own workspace. Always present. Can't be uninstalled.
2️⃣
Installed (node_modules scan)
Scan node_modules/ for packages with keyword core-shell-extension. Any npm package qualifies.
3️⃣
Local dev (extensions/ folder)
Scan extensions/ for subfolders with an index.tsx or package.json. Highest priority — overrides installed if IDs match. This is your dev loop.
