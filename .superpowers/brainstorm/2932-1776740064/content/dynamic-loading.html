Section 3 (cont.) — Dynamic Loading

Extensions load and reload at runtime. No restart required — for installs or for dev edits.

The reactive heart: ExtensionRegistry

Shell's central registry — a Zustand store the entire UI watches
// When registry changes, React re-renders the shell automatically.
// No manual wiring. No restart.

ExtensionRegistry = {
  extensions: {
    'amplifier-canvas': { manifest, status: 'active' },
    'git-panel':        { manifest, status: 'active' },
    'new-ext':          { manifest, status: 'loading' },  // ← appears live
  }
}

Two dynamic paths

Install from npm — no restart
1. User runs 
core-shell install @myorg/my-ext
↓
2. Main process
Runs npm install, then sends extension:installed IPC event to renderer
↓
3. Renderer
Dynamic import(extensionPath), reads manifest, adds to ExtensionRegistry
↓
4. Live — activity icon appears, canvas available
React re-renders shell. No restart.
Dev hot-reload — edit → live in ~1s
1. You edit a file
extensions/my-ext/Canvas.tsx saved
↓
2. chokidar detects change
Watching extensions/. Sends extension:changed IPC event.
↓
3. Renderer hot-swaps
Unmounts old, import(path + '?t=' + Date.now()) cache-busts, re-registers manifest
↓
4. Live — your edit is visible
Extension state optionally preserved via serialize/restore hooks.

Extension lifecycle hooks (optional)

extensions/my-ext/index.tsx — optional lifecycle
export const manifest = {
  id: 'my-ext',
  // ... contributes ...

  // Optional — called before unload (for hot-reload state preservation)
  onBeforeUnload: () => ({ savedState: myStore.getState() }),

  // Optional — called after reload with saved state
  onAfterLoad: (ctx) => {
    if (ctx.previousState) myStore.setState(ctx.previousState)
  },
}
