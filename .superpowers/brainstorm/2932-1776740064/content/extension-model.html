Section 3 — Extension Model

What an extension looks like from the inside. Minimal surface, maximum output.

Extension folder structure
extensions/ └─ amplifier-canvas/ ├─ index.tsx ← entry point ├─ Canvas.tsx ← main canvas UI ├─ Sidebar.tsx ← session list ├─ Panel.tsx ← terminal panel └─ StatusItem.tsx ← status bar
index.tsx — the entire extension contract
import { Canvas, Sidebar, Panel, StatusItem } from './components' import { SessionsIcon } from './icons' export const manifest = { id: 'amplifier-canvas', displayName: 'Amplifier Canvas', version: '0.1.0', contributes: { activityRail: { icon: SessionsIcon, title: 'Sessions' }, sidebar: Sidebar, canvas: Canvas, // ← the main UI panel: { title: 'Terminal', component: Panel }, statusBar: { position: 'left', component: StatusItem }, }, }
What the shell gives extensions — useShell()
// Inside any extension component: const shell = useShell() // ── State ────────────────────────────── shell.theme // 'light' | 'dark' shell.activeExtension // currently focused extension id // ── Navigation ───────────────────────── shell.openTab(extensionId) shell.setTitle(text) // update title bar // ── IPC bridge (→ main process) ──────── await shell.ipc.readFile(path) await shell.ipc.writeFile(path, content) shell.ipc.watchDir(path, callback) // → unsubscribe fn await shell.ipc.spawn(cmd, args) // → PTY instance // ── Commands ─────────────────────────── shell.registerCommand('canvas.clearAll', handler) shell.executeCommand('workbench.reload')
Extension-local state (no Zustand boilerplate)
// Each extension gets its own isolated store const [state, setState] = useExtensionStore({ sessions: [], activeSessionId: null, }) // Shell handles persistence, extension // doesn't need to think about it.
How extensions are discovered and loaded
1. Discovery
Shell scans extensions/ dir on startup. Any subfolder with an index.tsx is an extension candidate.
→
2. Import
Dynamic import() of the manifest. Components stay lazy until the user activates their slot.
→
3. Register
Contributions added to shell's slot registry. Activity rail icon appears. Commands registered in palette.
→
4. Mount
When user activates, React mounts the component inside an error boundary. Crash = scoped to that slot only.

New extension = new folder. No CLI. No publish. No config file outside the folder. Restart and it's there.
